Claude/Codex Hook 通知最终方案

当前代码先交付后端 MVP：agent-notify-tray 作为 localhost 后台自动检查并安装 Claude/Codex hooks，接收官方 hook 上报的事件，弹出 Windows Toast。完整 Tauri 托盘 UI、通知点击 deep link 和 PID/标题唤窗 fallback 仍是后续计划。

自动检查/安装 hooks Claude Code hook Codex lifecycle hooks 本地后台 MVP 离线事件丢弃 不自动批准权限

第一版承诺

2 AI CLI 都走官方 hook
0 离线事件补发次数
160 通知详情最大字符数
30s 同类通知去重窗口
当前实现：Axum localhost 后台、Bearer 鉴权、内存 session 表、30 秒去重、PowerShell Windows Runtime Toast、Hook Manager 自动安装/修复、HWND-only /focus/{sessionId}。未实现：完整 Tauri 托盘 UI、Toast 点击回调、agent-notify://focus?activationId=...、PID/进程树/窗口标题 fallback、session 详情页和 agentrun。

详细流程图

1

启动自检

agent-notify-tray serve 启动时检查 hooks；托盘监听入口待实现。

  • claude --version
  • codex features list
2

自动安装 Hook

缺失或过期时备份用户配置，再合并本项目 hook。

  • %LOCALAPPDATA%\AgentNotify\hooks
  • 不覆盖用户已有 hook
3

任意终端启动

用户仍按原习惯打开 Windows Terminal、PowerShell、CMD 或 IDE 终端。

  • claude
  • codex
4

官方 Hook 触发

Claude/Codex 在生命周期节点调用用户级 hook。

  • Claude Stop / Notification
  • Codex PermissionRequest / Stop
5

通用 Hook 入口

agent-notify-hook.ps1 读取 stdin payload、参数和环境变量。

  • 生成 sessionId
  • 提取 cwd / PID / 窗口信息
6

标准事件 JSON

把不同工具 payload 统一为同一种事件协议。

  • eventType
  • severity
  • message
7

CLI 转发

hook 不直接发 Toast，只调用：

agent-notify emit --stdin

  • 本地 token
  • 2 秒内退出
8

本地后台

当前 Axum 后台接收事件，更新内存 session 状态并去重。

  • 启动配置监听开关
  • 30 秒去重
  • 离线直接丢弃
9

Windows 通知

按通知策略展示简短、安全的气泡通知。

  • 点击回调待实现
  • 不显示完整命令
  • 不提供批准按钮
10

点击唤窗

当前仅支持 HTTP 调用 /focus/{sessionId} 后按 HWND 聚焦；Toast 点击和 PID/标题 fallback 待实现。

  • 独立窗口较稳
  • 多 tab 只保证 best-effort
监听关闭

事件直接丢弃

用户关闭监听、本地后台未运行、token 无效或本地通信失败时，不写离线队列，也不补发过期通知。

唤窗失败

打开 session 详情

session 详情页是后续 Tauri UI 降级；当前接口返回 openedSessionDetail 标志但不会打开 UI。

Codex Hook 映射

Codex lifecycle hook 统一事件 最终效果
SessionStart task.started 更新内存 session 列表，默认不弹通知；托盘列表待实现。
PermissionRequest user.confirmation_required 弹出高优先级通知；点击回到终端待实现。
Stop task.completed 或 user.input_required 提示本轮完成或等待下一步。
PostToolUse tool.blocked 或 task.failed 提示工具调用受阻或失败。

点击后的行为

  1. 当前 Toast 不携带点击回调。
  2. 可用 Bearer token 调用 POST /focus/{sessionId}。
  3. 后台查 session 表，并仅按 HWND 尝试定位终端。
  4. deep link、activation nonce、PID/标题 fallback 和 session 详情页待实现。

通知不会直接批准 Claude/Codex 的权限请求。用户仍必须回到终端内确认。
